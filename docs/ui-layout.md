# UI レイアウト（単一ページ・三カラム）

目的: 左=入力、中央=地図、右=候補フィード。カード選択と地図アノテーションを常時同期。

## レイアウト仕様

- 3カラム固定（PC前提）。幅比率 2:5:3 を基準。高さはビューポート100%。
- スクロールはページ単位ではなく、右カラム（候補フィード）のみ縦スクロール。
- ヘッダは置かない。画面占有を最大化。

## 左カラム（入力）

- セクション: 「GTFSを読み込む（.zip）」「優先KPIの選択（単一選択）」、プライマリボタン「シナリオ抽出」、セカンダリボタン「SDTで評価」。
- 優先KPI: カバー率 / 平均所要時間 / 運行コスト / サービスレベル維持率。
- ボタン文言は `labels-and-terminology.md` に準拠。

shadcn/ui 推奨コンポーネント:
- `Input` または `Dropzone` 相当（ファイル入力）
- `Select`（優先KPI 単一選択）
- `Button`（シナリオ抽出 / SDTで評価）

ガード:
- GTFS 未読込時は「シナリオ抽出」を非活性（disabled）
- 候補 0 件時は「SDTで評価」を非活性

## 中央カラム（地図）

- 初期表示: 現状ネットワーク（線=路線、点=停留所）＋利用状況の簡易ヒート。
- カード選択時: 対応するアノテーション（線/点/面）を強調表示（色/太さ/フィル）。
- フィルタ/ソート適用時: アノテーションの表示/非表示をカードと一致させる。

shadcn/ui 推奨コンポーネント:
- `Separator`（上部/下部の区切り 省略可）

## 右カラム（候補フィード）

- 常時 10 枚のカードを並べる（2列×5行 もしくは 1列縦並び）。
- 各カード: タイトル（候補ID）、1行理由、状態バッジ（未評価/評価中/達成/非達）、KPI 表（実測のみ）。
- 進捗/再送/最終更新はカード右上の小ボタン群に集約。

shadcn/ui 推奨コンポーネント:
- `Card`（候補表示の骨格）
- `Badge`（状態バッジ/推奨バッジ）
- `Tooltip`（推奨式/閾値出典、多様性フラグ）
- `Progress`（評価%）
- `Skeleton`（生成直後の骨格）

## アクセシビリティ

- フォーカス移動でカード→該当アノテーションへジャンプ（画面内スクロール併用）。
- ツールチップは hover / focus の双方で開く。
