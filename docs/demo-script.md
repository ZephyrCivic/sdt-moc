# デモ進行台本（想定ストーリー）

1. アプリ起動直後、地図に現状ネットワーク＋利用状況が表示されている。
2. 左カラムで `GTFSを読み込む` から ZIP を選ぶ。
3. 優先KPIを選ぶ（例: カバー率）。
4. 「シナリオ抽出」を押す → 右カラムに 10 枚のカード（未評価/KPI非表示/1行理由）。
4. 任意のカードをクリック → アノテーション（線/点/面）が地図でハイライト。
5. 「SDTで評価」を押す → 全カードが評価中に遷移、進捗%・最終更新・再送が現れる。
6. 評価完了したカードから実測KPIが表示、達成/非達のバッジが更新される。
7. 上位候補に「推奨」バッジが付く。トグル「推奨のみ表示」で絞り込み。
8. 優先KPIを別軸に切り替える → KPI値は固定のまま、並びと推奨が即時に更新される。
9. タイムアウト発生例を示し、自動→手動再送で復旧できることを確認。
10. 受入チェックリストに沿って完了条件を読み上げ、デモ終了。
