# 推奨ロジック（優先KPI × 補助KPI）

目的: 優先KPIを第一基準、補助KPIは閾値確認のみ。ブラックボックス感を減らすため、式と出典をUIで開示。

## 定義

- 優先KPI: ユーザーが単一選択（左カラム）。
- 補助KPI: 残り3軸。しきい値を満たすことを確認（劣後でありソートには影響しない）。

## 判定

1) `achieved` のみ母集団とする（未評価/評価中は対象外）。
2) 優先KPIのソート順に上位を推奨候補とする（同値は補助KPIの合議順で安定化）。
3) 推奨数は UI 上は最大3件を強調（バッジ）し、トグル「推奨のみ表示」で絞り込める。

## UI開示（ツールチップ）

- 簡易式: 例）`推奨条件 = coverageRate 上位 & 他3軸 >= 閾値`
- 閾値出典: 例）`社内ガイドライン vX.Y（YYYY-MM-DD）`

詳細は `kpis.md`, `filters-and-sorting.md` を参照。
